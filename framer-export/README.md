# Framer import set

Generated by `workflow/build_framer_import.py` from the `blog/*/draft.md`
drafts. Re-run that script any time a draft changes to refresh these files.

```
framer-export/
├── framer-import.csv   one row per post, columns = CMS fields
└── bodies/<slug>.md    cleaned markdown body for each post (paste target)
```

## Recommended workflow (easiest, highest fidelity)

The fastest reliable path is **CSV import for the fields + markdown paste for
the body**. CSV handles all the metadata in one shot; pasting the body lets
Framer build the formatted blocks itself.

### 1. Create the CMS collection in Framer

In the Framer project ([open in Framer](https://framer.com/projects/PSFnetwork--6bKb0EpuKuOgXIlv9Apl-640G6)),
open **CMS → New Collection → "Blog"** and add fields matching the CSV columns:

| CSV column | Framer field type |
|------------|-------------------|
| `slug` | Slug |
| `title` | Plain text (SEO / meta title) |
| `display_title` | Plain text (on-page H1) |
| `dek` | Plain text (subtitle) |
| `type`, `topic` | Plain text or Option |
| `author`, `reviewer` | Plain text (or Reference to an Authors collection) |
| `read_time` | Plain text |
| `published`, `updated` | Date |
| `focus_keyword`, `secondary_keywords` | Plain text |
| `meta_description` | Plain text |
| `canonical` | Link |
| `hero_visual_alt` | Plain text (alt for the hero Image field) |
| `content` | Formatted text (rich text) |

### 2. Import the CSV

CMS collection → **`···` menu → Import → CSV** → pick `framer-import.csv`.
Map each column to its field. This populates every field, including a
plain-text copy of `content`.

### 3. Paste each body for clean formatting

CSV import drops the body in as plain text. To get real headings/lists/links,
open each item's **Formatted text** field, then copy the matching
`bodies/<slug>.md` file and paste it in. Framer converts the markdown into
formatted blocks automatically. (You can skip this for posts where the
plain-text import is good enough, but pasting gives the best result.)

### 4. Hero images

Bodies have no hero image, so add it to the collection's **Image** field per
post; `hero_visual_alt` already carries the alt text.

## Notes

- `display_title` (body H1) and `title` (frontmatter SEO title) intentionally
  differ per post; both are preserved so the page H1 and meta title can diverge.
- Internal `Related` links use root-relative paths like `/blog/<slug>` and will
  resolve once the collection's detail page slug pattern is `/blog/:slug`.
